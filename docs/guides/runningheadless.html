Installing Fiji

  • Linux
  • Mac
  • Windows 
# Commands for Bash
# Downloading the latest copy of Fiji for 64-bit Linux 
wget https://downloads.imagej.net/fiji/latest/fiji-linux64.zip

# Unzipping archive
unzip fiji-linux64.zip

# Removing downloaded zip archive
rm fiji-linux64.zip

# Moving into extracted directory
cd Fiji.app/
# Commands for Bash
# Downloading the latest copy of Fiji for 64-bit MacOS
curl -O https://downloads.imagej.net/fiji/latest/fiji-macosx.zip

# Unzipping archive
unzip fiji-macosx.zip -d 

# Removing downloaded zip archive
rm fiji-macosx.zip

# Moving into extracted directory
cd Fiji.app/Contents/MacOS
# Commands for Windows PowerShell
# Downloading the latest copy of Fiji for 64-bit Windows
Invoke-WebRequest https://downloads.imagej.net/fiji/latest/fiji-win64.zip

# Unzipping archive
Expand-Archive .\fiji-win64.zip

# Removing downloaded zip archive
Remove-Item .\fiji-win64.zip

# Moving into extracted directory
cd .\fiji-win64\Fiji.app

# Commands for Bash
# Marking MIA and IJPB-plugins for installation
./ImageJ-linux64 --headless --update add-update-site MIA https://sites.imagej.net/ModularImageAnalysis/
./ImageJ-linux64 --headless --update add-update-site IJPB-plugins https://sites.imagej.net/IJPB-plugins

# Commands for Bash

./ImageJ-macosx --headless --update add-update-site IJPB-plugins https://sites.imagej.net/IJPB-plugins

# Performing update
./ImageJ-macosx --headless --update update
# Commands for Windows PowerShell
# Marking MIA and IJPB-plugins for installation
.\ImageJ-win64 --headless --console --update add-update-site MIA https://sites.imagej.net/ModularImageAnalysis/

# Performing update
.\ImageJ-win64 --headless --console --update update

Running a workflow

  • Linux
  • Mac
  • Windows 
# Commands for Bash
# (Optional) Download example workflow to user folder
wget https://github.com/mianalysis/mia-examples/archive/refs/tags/v1.0.0.zip -O ~/example.zip
unzip ~/example.zip -d ~/
rm ~/example.zip

# Defining workflow path as a variable
wfPath=~/mia-examples-1.0.0/Ex1_NucleiSegmentation/Ex1_NucleiSegmentation.mia

# Running a workflow on a specified single file
inPath=~/mia-examples-1.0.0/Ex1_NucleiSegmentation/Ex1_3DStack1.tif
./ImageJ-linux64 --headless --run "MIA (headless)" "workflowPath='$wfPath', inputPath='$inPath'"

# Running a workflow on a specified folder with verbose output
inPath=~/mia-examples-1.0.0/Ex1_NucleiSegmentation/
./ImageJ-linux64 --headless --run "MIA (headless)" "workflowPath='$wfPath', inputPath='$inPath', verbose=true"

Arguments

Argument Required? Default value Description
workflowPath Required N/A Path to .mia workflow file. This can either be relative to the current directory or an absolute system path.
inputPath Optional [Blank] If set, this will override the "Input path" file/folder set in the specified workflow. When left blank, the "Input path" of the workflow file will be used.
variables Optional [Blank] Global variables (i.e. those specified as name-value pairs in a "Global Variables" module) can be overridden using the form "name1:value1;name2:value2;etc.".
showDebug Optional false Display any debug messages encountered during workflow execution to the terminal. Debug messages are displayed in blue.
showMemory Optional false Following execution of each module the current memory usage and elapsed time are displayed in the terminal. Memory messages are displayed in green.
showMessage Optional true Display any general messages encountered during workflow execution to the terminal. Debug messages are displayed in white.
showStatus Optional true Display module status updates in the terminal. Status messages are displayed in blue.
showWarning Optional true Display any warning messages encountered during workflow execution to the terminal. Warning messages are displayed in orange.
verbose Optional false If processing multiple files/series (i.e. "batch mode"), verbose mode will display the current module status, rather than the number of jobs completed.
